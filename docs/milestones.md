# Doctor Secretary AI - Milestones (90 Days)

## Weeks 1-2: Foundations
- Finalize workflows, intent taxonomy, and emergency rules.
- Confirm data model and record fields.
- Define approval and escalation policies.
- Draft initial UI for web dashboard.

## Weeks 3-5: Messaging + Admin Flow
- Implement WhatsApp adapter (Baileys for internal testing only).
- Message normalization and triage pipeline.
- Admin workflows: appointment request, confirm, reschedule, cancel.
- Quick-reply approval commands in WhatsApp.

## Weeks 6-8: Documents + Records
- Document ingestion (PDF/JPEG/PNG).
- OCR pipeline and storage.
- Patient record viewer and search.
- Summarization of multi-document history.

## Weeks 9-10: Prescriptions
- Prescription draft workflow.
- Template-based PDF output.
- Bilingual instructions (English + Hindi rules-based).

## Week 11: Web Dashboard
- Records and document viewer.
- Prescription composer.
- Audit log and approvals view.

## Week 12: Payments
- UPI link generation and status tracking.
- Receipts and appointment confirmation.

## Week 13: Pilot Readiness
- Internal QA and safety checks.
- Pilot with 3-5 doctors.
- Collect feedback, fix workflow gaps.

## Pre-Launch Checklist
- Implement official WhatsApp Business adapter.
- Security review and data retention controls.
- Configure emergency templates per doctor.
- Operational playbook for support issues.
